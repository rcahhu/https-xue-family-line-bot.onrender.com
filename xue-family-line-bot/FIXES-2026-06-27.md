# 2026-06-27 修正版重點

這版主要處理 LINE 官方帳號實測時常見的流程問題。

## 已修正

1. `功能`、加入好友、加入群組會回傳功能 Flex 選單，不再只回文字。
2. 圖文選單常用文字已補齊：
   - `建立旅遊日記` / `新增日記` / `建立新日記`
   - `打開日記` / `開啟日記` / `打開日記頁`
   - `熱門景點` / `熱門美食` / `附近推薦`
3. 功能選單裡的「建立旅遊日記」會開啟 `/app?new=1` 的新日記頁；若設定 LIFF_ID，會用 LIFF 連結開啟。
4. 功能選單裡的「打開日記」若設定 LIFF_ID，會用 LIFF 連結開啟。
5. Webhook 改成逐筆處理 LINE events，單筆失敗不會讓整個 webhook 回 500，降低 LINE 重送造成重複寫入的機率。
6. JSON 檔案儲存加入寫入佇列，避免多個新增/修改同時進來時互相覆蓋。
7. `/app/` 尾斜線也會正確開啟首頁，避免部署平台或 LIFF endpoint 多一個斜線時 404。
8. 訪客模式在 localStorage 或 crypto 受限的瀏覽器中不會直接白畫面。
9. OpenAI 預設模型範例由 `gpt-5.4-mini` 改為 `gpt-5-mini`。

## 已測試

- `node --check`：server、LINE、AI、storage、diary import、前端 app.js 均通過。
- 既有 smoke test 通過。
- 既有 feature test 通過。
- 追加測試：`功能`、`建立日記`、`打開日記`、`熱門景點` 不會被錯誤寫成行程。
